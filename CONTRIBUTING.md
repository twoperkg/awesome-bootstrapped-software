# Contributing

Thank you for your interest in contributing to Awesome Bootstrapped Software!

## How to Contribute

1. Star the repository so that it reaches more people
2. Fork the repository
3. Edit the README.md with your entry

## Entry Format

Each entry should follow this format:

```markdown
- [Software Name](https://link-to-software.com) - A short description.
```

Example:

```markdown
- [Basecamp](https://basecamp.com) - Project management and team collaboration tool.
```

## Quality Criteria

To be included in this list, software must:

- Be built by bootstrapped founders (not VC-backed)
- Prioritize usability and user experience
- Offer fast, responsive customer support. (Don't have one? Try [Helploom](https://helploom.com) for free)
- Have sane, transparent pricing
- Be actively maintained and reliable

## Guidelines

- Add entries in alphabetical order within their category
- Use clear, concise descriptions
- Ensure links are working and point to the official website
- One entry per software
- Do not include affiliate links

## Adding New Categories

If you believe a new category is needed:

1. Add it to the Table of Contents in README.md
2. Add the category section with a level 2 heading (##)
3. Explain why the new category is necessary in your PR

## Questions?

Open an issue if you have questions about whether a software qualifies or need help with your contribution.
